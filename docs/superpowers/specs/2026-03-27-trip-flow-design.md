# Flow "Собираюсь в горы" — Design Specification

## Overview

Центральный пользовательский сценарий приложения Mountaine — от решения ехать в горы до выхода на маршрут. Объединяет модули: маршруты, кладовка, сборы.

## Точки входа (3 входа → один flow)

- **Главная:** кнопка "Собираюсь в горы" → визард с шага 1 (регион)
- **Страница маршрута:** кнопка "Собираться" → визард с маршрутом уже выбранным (шаг 4 предзаполнен)
- **Кладовка:** кнопка "Создать сборы" → визард с шага 1

## Шаги визарда

### Шаг 1. Выбор региона
Карточки доступных регионов (MVP: Хибины). Каждая карточка: название, фото, количество маршрутов, диапазон категорий.

### Шаг 2. Шаблон сборки
4 карточки с нарастающим набором снаряжения:
- **Лёгкий треккинг** (~8кг) — базовая одежда, рюкзак 30л, палки, фонарь, бивуак лёгкий
- **НП** (~15кг) — + обвязка, каска, верёвка, карабины, базовое железо
- **СП-3** (~18кг) — + кошки, ледоруб, ледобуры, больше железа, бивуак зимний
- **СП-2 и выше** (~22кг) — + ледовые инструменты, полный набор закладок/френдов, ИТО

Каждый шаблон включает предыдущий + дополнительное снаряжение.

**Логика:** шаблон автоматически заполняет набор сборов из кладовки пользователя. Что есть — добавляется. Чего нет — помечается как "нужно приобрести/взять".

### Шаг 3. Настройка снаряжения
Список из шаблона с цветовой индикацией:
- Зелёные — есть в кладовке, добавлено в набор
- Красные — нет в кладовке
- Можно добавить/убрать вручную
- Распределение по рюкзакам
- Вес в реальном времени

### Шаг 4. План маршрутов (опционально)
- Список маршрутов выбранного региона с фильтром по категории
- Выбрать один или несколько маршрутов
- Или кнопка "Решу на месте" → пропустить

### Шаг 5. "На маршрут" — полноэкранный режим
Вкладки:
- **Маршрут** — описание, станции R0-R1..., категория
- **Снаряжение** — список по рюкзакам + общий вес
- **Чеклист** — упаковано, документы, связь, страховка
- **Экстренные контакты** — МЧС, координаты, телефоны

**Оффлайн:** возможность сохранить для работы без связи (кэш + PDF-экспорт).

## Индикатор активных сборов

Маленький бейдж в навбаре на всех страницах: "Хибины, март" → клик → страница /trips с полной информацией.

## Страница /trips

Все поездки пользователя:
- Активные (в процессе сборов)
- Запланированные
- Прошлые

## Модель данных

Новая таблица `trips`:
- id, user_id, name, region (mountain_id), status (planning/packing/active/completed)
- template (light_trek/np/sp3/sp2)
- packing_set_id (FK → packing_sets)
- created_at, updated_at

Таблица `trip_routes`:
- id, trip_id, route_id

## Tech Stack

Используется существующая инфраструктура: Next.js, Supabase, Tailwind. Новых библиотек не требуется. Оффлайн — через Service Worker + PDF-генерацию.
